Есть компонент с кнопкой, которая увеличивает счётчик. Используется setTimeout внутри обработчика клика:
```jsx
function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setTimeout(() => {
      setCount(count + 1);
    }, 1000);
  };

  return (
    <div>
      <p>Счётчик: {count}</p>
      <button onClick={handleClick}>+1 через 1 сек</button>
    </div>
  );
}
```
count попадает в замыкание в момент клика. Если нажать кнопку несколько раз быстро, каждый setTimeout использует одно и то же старое значение
Исправить баг, из-за которого в setTimeout используется устаревшее значение состояния.

```jsx

  console.log("Child render");

  return <button onClick={onAction}>Child button</button>;
});

function Parent() {
  const [count, setCount] = useState(0);

  // Новая функция создаётся при каждом рендере
  const handleClick = () => {
    console.log("Clicked");
  };

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Parent count: {count}</button>

      <Child onAction={handleClick} />
    </div>
  );
}
```
Найдите причину и исправьте её с помощью useCallback
